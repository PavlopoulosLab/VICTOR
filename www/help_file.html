This tab allows the user to upload files containing clustering results. Once uploading has been completed, files can be renamed or removed. Any additional uploaded files are appended to the list. For selected files, basic statistics such as the number of clusters and objects are visualized as bar charts, whereas histograms show the object participation in the various clusters. Object filtering options are also available.

1. The Upload Multiple clustering Files option allows the user to upload clustering result files in text format. The file must consist of 2 columns without headers. Each object must belong to only one cluster (no duplicates allowed). The first column contains the cluster names, whereas the second column the cluster members in a comma-separated format. After the files have been uploaded, the user has the option to delete, rename and upload more files.

2.The Cluster Object Cutoff Range option allows the user to filter clusters of selected filess, by applying minimum and maximum thresholds on the number of participating objects per cluster.

3.The third option allows the user to filter the clustering files by selecting either their object intersection or by creating singleton clusters for missing objects. This step is required when the selected clustering files for analysis (Compare Clusterings) do not contain exactly the same objects. The Intersection filtering option retains only objects that are common between all of the selected clustering files. On the other hand, for every object that is present in a clustering file but is missing from others, the Superset filtering option constructs a singleton cluster and assigns it to the latter.

Clustering file format

    cluster1	F19,F21,F22,F27,F29,F33,F41,F42,F44,F47
    cluster2	F02,F10,F13,F15,F18,F24,F25,F26,F46,F50
    cluster3	F01,F12,F35,F40,F45
    cluster4	F03,F04,F06,F07,F08,F17,F20,F36,F38,F43,F49
    cluster5	F05,F09,F16,F28,F30,F32,F34,F39
    cluster6	F11,F14,F23,F31,F37,F48

This is an example of the required input file format used in VICTOR.

Selected files can be removed or renamed via the designated buttons.

Bar charts

Any selected raw or filtered files are visualized as bar charts. The upper part of the figures show the number of clusters per clustering file. The bottom bar plots show the total number of objects per clustering. On the left figure we observe that cluster 7 does not contain the same number of objects with the other files. After filtering the data with the Superset option, all clusterings contain exactly the same objects (right figure) and are ready to be analysed (Compare Clusterings).

Participation Histograms

Participation histograms of both raw and filtered clustering files show the object participation for a user-selected clustering file.

Data

A selected clustering file in its raw or filtered version can be visualized as a table. The first column shows the cluster names, the second the cluster objects and the third the number of objects in each cluster. The user can download the clustering data in various formats.
